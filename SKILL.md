# Error Journal - 错误日志技能

## 概述

这个技能帮助 Claude Code 记录执行 bash 命令时遇到的错误，形成可检索的错误知识库。

## 何时使用

1. **自动触发**：当 bash 命令执行失败（返回非零退出码）时
2. **手动触发**：用户说 "记下来这个错误"、"这个坑要记住"、"记录错误" 时
3. **事后记录**：解决问题后，用户要求记录教训时

## 错误日志位置

```
~/.claude/skills/error-journal/references/error-log.md
```

## 记录格式

每条错误记录使用以下格式：

```markdown
## [#序号] 简短标题

| 字段 | 内容 |
|------|------|
| **日期** | YYYY-MM-DD HH:MM |
| **命令** | `执行的命令` |
| **错误** | 错误信息摘要 |
| **上下文** | 当时在做什么 |
| **根因** | 为什么出错 |
| **方案** | 如何解决的 |
| **教训** | 以后如何避免 |

---
```

## 工作流程

### 1. 错误发生时

当 bash 命令失败时：

1. 读取 `references/error-log.md` 获取当前序号
2. 分析错误原因
3. 追加新记录到日志末尾
4. 简短告知用户 "已记录此错误"

### 2. 查询历史错误

当用户问 "有没有遇到过..."、"之前是不是..." 时：

1. 读取错误日志
2. 搜索相关关键词
3. 返回匹配的历史记录

### 3. 解决问题后补充

如果一开始只记录了错误，解决后：

1. 找到对应的错误记录
2. 补充 "方案" 和 "教训" 字段

## 使用示例

### 示例 1：自动记录

```
[Claude 执行命令]
ssh root@server "docker-compose up -d"

[终端返回]
Error: port 80 already in use

[Claude 自动记录]
## #3 Docker 端口占用

| 字段 | 内容 |
|------|------|
| **日期** | 2024-01-15 14:30 |
| **命令** | `docker-compose up -d` |
| **错误** | port 80 already in use |
| **上下文** | 部署服务到生产服务器 |
| **根因** | (待分析) |
| **方案** | (待解决) |
| **教训** | (待总结) |
```

### 示例 2：手动触发

```
用户: 这个坑要记住，npm install 时要用 --legacy-peer-deps

[Claude 记录]
## #4 npm 依赖冲突

| 字段 | 内容 |
|------|------|
| **日期** | 2024-01-15 15:00 |
| **命令** | `npm install` |
| **错误** | peer dependency conflict |
| **上下文** | 安装项目依赖 |
| **根因** | npm 7+ 严格检查 peer dependencies |
| **方案** | 使用 `npm install --legacy-peer-deps` |
| **教训** | 遇到 peer dep 冲突时，先尝试 --legacy-peer-deps |
```

### 示例 3：查询历史

```
用户: 之前遇到过端口占用的问题吗？

[Claude 搜索日志]
找到 #3 Docker 端口占用，当时是...
```

## 最佳实践

1. **及时记录**：错误发生后立即记录，记忆最清晰
2. **补充完整**：问题解决后，回去补全 "方案" 和 "教训"
3. **简洁明了**：错误信息摘要即可，不需要完整堆栈
4. **关键词友好**：标题和教训使用容易搜索的关键词

## 反模式（避免）

- 记录太详细，导致日志臃肿
- 只记录不补充，缺少解决方案
- 记录无关紧要的小错误
